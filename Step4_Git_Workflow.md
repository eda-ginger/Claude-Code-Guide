# Step 4: Git 워크플로 가이드

이 문서에서는 Claude Code를 활용한 **Git 커밋, 브랜치 관리, Pull Request 생성, 코드 리뷰** 등 실전 Git 워크플로를 안내합니다.

## 1. 커밋 만들기

Claude Code에게 커밋을 요청하면, 변경 사항을 분석하여 적절한 커밋 메시지를 자동으로 작성합니다.

```
> 지금까지 변경한 내용을 커밋해줘
```

또는 `/commit` 슬래시 명령을 사용할 수 있습니다.

```
> /commit
```

Claude는 다음을 자동으로 수행합니다:
1. `git status`와 `git diff`로 변경 사항 확인
2. 최근 커밋 히스토리를 참고하여 스타일에 맞는 메시지 작성
3. 관련 파일만 선별하여 `git add` 후 커밋

### 1.1 커밋 메시지 규칙

Claude Code는 프로젝트의 기존 커밋 스타일을 따릅니다. 원하는 규칙이 있다면 CLAUDE.md에 명시하세요.

```markdown
## 커밋 컨벤션
- feat: 새로운 기능
- fix: 버그 수정
- docs: 문서 변경
- refactor: 리팩토링
```

## 2. 브랜치 관리

```
> feature/login 브랜치를 만들고 전환해줘
> 현재 브랜치를 main으로 리베이스해줘
> 이 브랜치의 커밋 히스토리를 보여줘
```

## 3. Pull Request 생성

Claude Code는 `gh` CLI를 활용하여 GitHub PR을 직접 생성할 수 있습니다.

```
> 이 브랜치로 PR을 만들어줘
```

Claude는 다음을 수행합니다:
1. 현재 브랜치의 모든 커밋과 변경 사항 분석
2. PR 제목과 본문 자동 작성 (Summary + Test plan 포함)
3. 리모트에 push 후 `gh pr create` 실행

### 3.1 PR 생성 전 준비사항

- **GitHub CLI (`gh`)가 설치 및 인증**되어 있어야 합니다.
  ```bash
  # 설치 확인
  gh --version

  # 인증 (최초 1회)
  gh auth login
  ```

## 4. 코드 리뷰

기존 PR의 코드를 리뷰하도록 요청할 수도 있습니다.

```
> PR #42를 리뷰해줘
> https://github.com/owner/repo/pull/42 리뷰해줘
```

Claude는 PR의 변경 사항을 분석하고 다음을 확인합니다:
- 코드 품질 및 잠재적 버그
- 보안 취약점
- 성능 이슈
- 코딩 컨벤션 준수 여부

## 5. 충돌 해결

머지 충돌이 발생했을 때도 Claude Code에게 도움을 받을 수 있습니다.

```
> 현재 머지 충돌을 해결해줘
> main 브랜치와의 충돌을 보여주고 해결 방법을 제안해줘
```

## 6. Git 워크플로 실전 예시

전체 흐름을 하나의 시나리오로 보면 다음과 같습니다:

```
# 1. 새 브랜치 생성
> feature/add-login 브랜치를 만들어줘

# 2. 코드 작성
> 로그인 페이지 컴포넌트를 만들어줘

# 3. 테스트 작성 및 실행
> 로그인 컴포넌트의 테스트를 작성하고 실행해줘

# 4. 커밋
> 변경사항을 커밋해줘

# 5. PR 생성
> main 브랜치로 PR을 만들어줘
```

---
> 💡 **Tip:** Claude Code는 `git push --force`나 `git reset --hard` 같은 위험한 명령을 실행하기 전에 반드시 사용자에게 확인을 요청합니다. 안심하고 사용하세요.
