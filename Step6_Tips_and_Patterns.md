# Step 6: 실전 팁과 활용 패턴

이 문서에서는 Claude Code를 더 효율적으로 사용하기 위한 **프롬프트 패턴, 토큰 절약법, 트러블슈팅** 등 실전 노하우를 안내합니다.

## 1. 효과적인 프롬프트 작성법

### 1.1 구체적으로 요청하기

Claude Code는 구체적인 지시일수록 정확한 결과를 반환합니다.

| 비효율적 | 효율적 |
|----------|--------|
| "이 코드 고쳐줘" | "login.js의 handleSubmit에서 빈 이메일일 때 에러 메시지를 표시해줘" |
| "테스트 만들어줘" | "UserService.create 메서드에 대해 유효/무효 입력 케이스의 단위 테스트를 작성해줘" |
| "성능 개선해줘" | "getUsers API가 N+1 쿼리 문제가 있는 것 같은데 확인하고 수정해줘" |

### 1.2 단계적으로 요청하기

큰 작업은 한 번에 요청하기보다 단계별로 나누면 품질이 높아집니다.

```
# 1단계: 현황 파악
> 현재 인증 시스템의 구조를 분석해줘

# 2단계: 계획 수립
> OAuth 2.0으로 전환하려면 어떤 파일들을 수정해야 하는지 계획을 세워줘

# 3단계: 구현
> 계획대로 1번 항목부터 구현해줘
```

### 1.3 컨텍스트 제공하기

배경 정보를 함께 제공하면 더 적합한 결과를 얻을 수 있습니다.

```
> 우리 팀은 React + TypeScript를 사용하고, 상태 관리는 Zustand를 쓰고 있어.
> 사용자 프로필 편집 페이지를 만들어줘.
```

## 2. 토큰 절약 방법

Claude Code는 API 토큰을 소비하므로 효율적으로 사용하는 것이 중요합니다.

### 2.1 `/compact` 활용

대화가 길어지면 `/compact` 명령으로 컨텍스트를 압축합니다.

```
> /compact
```

커스텀 요약 지시도 가능합니다:

```
> /compact 인증 관련 작업 내용만 유지해줘
```

### 2.2 `/clear`로 새 시작

완전히 다른 주제로 넘어갈 때는 `/clear`로 컨텍스트를 비웁니다.

### 2.3 CLAUDE.md 활용

반복적으로 설명하는 프로젝트 정보는 CLAUDE.md에 기록하면 매번 토큰을 소비하지 않아도 됩니다.

### 2.4 `/cost`로 사용량 모니터링

```
> /cost
```

현재 세션의 토큰 사용량과 비용을 확인할 수 있습니다.

## 3. 자주 쓰는 활용 패턴

### 3.1 탐색 → 이해 → 수정 패턴

새로운 코드베이스를 다룰 때 유용한 3단계 패턴입니다.

```
> 이 프로젝트의 전체 구조와 주요 진입점을 알려줘    # 탐색
> 결제 처리 로직이 어떻게 동작하는지 설명해줘       # 이해
> 결제 실패 시 재시도 로직을 추가해줘              # 수정
```

### 3.2 테스트 주도 수정 패턴

```
> 현재 실패하는 테스트를 보여줘
> 실패 원인을 분석해줘
> 테스트가 통과하도록 코드를 수정해줘
> 다시 테스트를 실행해서 확인해줘
```

### 3.3 코드 리뷰 패턴

```
> 이 PR의 변경사항을 보안 관점에서 리뷰해줘
> 성능 이슈가 있는지도 확인해줘
> 발견된 문제점에 대한 수정을 제안해줘
```

### 3.4 디버깅 패턴

```
> 이 에러 메시지를 분석해줘: [에러 내용 붙여넣기]
> 관련 코드를 찾아서 원인을 파악해줘
> 수정하고 동일 시나리오에서 문제가 해결되는지 확인해줘
```

## 4. 트러블슈팅

### 4.1 "Permission denied" 에러

Claude Code가 도구 사용 권한이 없다는 에러를 표시할 때:

```bash
# settings.json에서 해당 도구를 allow 목록에 추가
# 또는 실행 시 권한 플래그 사용
claude --allowedTools Bash,Read,Edit
```

### 4.2 컨텍스트 초과

대화가 너무 길어져 컨텍스트 한계에 도달하면:
- `/compact`로 압축하거나
- `/clear` 후 필요한 맥락만 다시 전달

Claude Code는 자동으로 이전 메시지를 압축하기도 합니다.

### 4.3 Claude Code 업데이트

```bash
npm update -g @anthropic-ai/claude-code
```

### 4.4 네트워크 문제

프록시 환경에서는 환경 변수를 설정합니다:

```bash
export HTTPS_PROXY=http://proxy.company.com:8080
```

## 5. 모델 선택

Claude Code에서 사용할 모델을 선택할 수 있습니다.

| 모델 | 특징 |
|------|------|
| **Opus** | 가장 높은 품질, 복잡한 추론에 적합 |
| **Sonnet** | 속도와 품질의 균형, 일반 코딩 작업에 적합 |
| **Haiku** | 가장 빠르고 저렴, 간단한 작업에 적합 |

모델 전환은 `/model` 명령이나 설정에서 가능합니다.

---
> 💡 **Tip:** Claude Code를 처음 사용한다면, 간단한 작업부터 시작하여 점차 복잡한 작업으로 확장하는 것을 추천합니다. 도구의 동작 방식과 권한 체계를 이해하고 나면 훨씬 효율적으로 활용할 수 있습니다.
