# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 프로젝트 개요

Claude Code의 다양한 기능을 실습하고 테스트하기 위한 **한국어 가이드 문서 저장소**입니다. 소스 코드가 아닌 마크다운 문서(.md)로 구성됩니다.

## 문서 구조

- `README.md` — 프로젝트 소개 및 가이드 목차
- `docs/Part1-Basic/` — 기본 가이드 (읽고 이해하는 이론 문서)
- `docs/Part2-Setup/` — 실전 환경 구축 (Claude에게 보여주면 세팅+활용까지 안내)
- `docs/Part3-Research/` — 연구·학습 활용 (연구 워크플로에 Claude Code 활용)
- `docs/Part4-Life/` — 일상·생활 활용 (Obsidian, 한국 생활 스킬 등)
- `docs/Part5-Appendix/` — 부록 (상세 레퍼런스, 보충 문서)

## 파일 명명 규칙

- 폴더명: `Part{번호}-{영문}` (예: `Part1-Basic`, `Part2-Setup`)
- 파일명: `{NN}-{영문키워드}.md` (예: `01-start.md`, `02-project.md`)
- 소문자, 하이픈(`-`) 구분, 언더스코어(`_`) 사용 금지
- 새 가이드 추가 시 반드시 `README.md`의 해당 Part 테이블에 링크 추가

## 문서 작성 규칙

### 공통 규칙

- 모든 가이드는 **한국어**로 작성
- H1(`#`)은 문서당 하나, 한국어 제목
- H1 바로 아래에 **한 줄 요약** 작성
- H1 아래 `---` 구분선으로 본문 시작
- 섹션 번호: `## 1.`, `## 2.` 형식 (하위 번호 `### 1.1` 사용하지 않음)
- 코드 블록에는 언어 태그 필수 (```bash, ```json, ```python 등)
- 마지막에 `> 💡 **Tip:**` 형식의 마무리 팁 1개
- 이모지는 사용자가 명시적으로 요청한 경우에만 사용

### 이론 문서 (Part1) — 사람이 읽고 이해하는 문서

```markdown
# 제목

한 줄 요약.

---

## 1. 섹션 제목

설명 텍스트. 개념을 먼저 설명하고, 예시를 보여준다.

| 항목 | 설명 |
|------|------|
| A | 설명 A |

## 2. 다음 섹션

...

---
> 💡 **Tip:** 마무리 팁.
```

**이론 문서 원칙:**
- 개념 설명 → 예시 → 비교 테이블 순서
- 실행 명령어보다 "왜 이렇게 하는지" 설명에 집중
- 한 섹션은 스크롤 없이 읽을 수 있는 분량 (20줄 이내) 권장

### 실행 문서 (Part2~4) — Claude에게 보여주면 수행하는 문서

```markdown
# 제목

> 한 줄 요약 — 이 문서를 Claude에게 보여주면 무엇이 완성되는지

---

## 사전 요구사항

| 도구 | 용도 | 확인 방법 |
|------|------|-----------|
| 도구A | 용도 | `command --version` |

## 1. 설치/설정

```bash
구체적인 명령어
```

## 2. 사용법

| 질문 방식 | 예상 결과 |
|-----------|-----------|
| `자연어 질문 예시` | 기대되는 출력 형태 |

## 트러블슈팅

| 증상 | 해결 |
|------|------|

---
> 💡 **Tip:** 마무리 팁.
```

**실행 문서 원칙:**
- 명령어는 복사-붙여넣기 가능한 완전한 형태로 작성
- "왜"보다 "무엇을, 어떻게" 중심
- 사전 요구사항을 반드시 먼저 명시
- 트러블슈팅 섹션 필수 포함
- 테스트 완료된 기능에는 질문 방식 + 예상 결과 테이블 포함

### 부록 문서 (Part5) — 상세 레퍼런스

- 메인 가이드에서 참조 링크로 연결
- 분량 제한 없음 (레퍼런스 성격)
- 예제 스크립트, JSON 필드 목록, 상세 설정 등 포함 가능

## 문서 품질 기준

- 한 문서는 **200줄 이하** 권장. 초과 시 분리 또는 Part5로 상세 내용 이동 검토
- 다른 문서와 **내용 중복 금지**. 중복 시 한쪽에서 참조 링크로 대체
- 테이블은 3열 이내로 간결하게. 복잡한 정보는 리스트로 전환
- 섹션 간 논리적 흐름: 개념 → 설정 → 사용 → 문제 해결
