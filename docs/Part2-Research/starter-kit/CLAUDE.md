# Mini Review Project: LLM-Based Code Generation

## Project Purpose
A mini review comparing three approaches to LLM-based code generation.
Target: Short review paper (5~10 pages), article format.

## Project Structure
- `primary/` — Immutable source documents (NEVER edit or delete)
  - `paper-list.md` — Final paper list from collection phase
  - `papers/` — Source paper PDFs (3 papers)
- `secondary/` — LLM-managed knowledge base (wiki)
  - `index.md` — Wiki index (updated on every ingest)
  - `log.md` — Operation log (append-only)
  - `summaries/` — Per-paper summary pages
  - `concepts/` — Cross-paper concept pages
  - `paper_comparison.md` — Comparison table (SSOT, generated by query)
  - `outline.md` — Review outline (generated by query)
- `work_paper/` — Active manuscript
  - `template/` — OUP journal template (copy to main.tex before editing)
  - `main.tex` — LaTeX manuscript (created from template, active editing target)
  - `figures/` — Generated figures (PDF/PNG)
- `CLAUDE.md` — This file (project rules)

## Knowledge Base Rules (Karpathy LLM-Wiki)

### Ingest Workflow
1. Read papers in `primary/papers/`
2. Create per-paper summary in `secondary/summaries/`
3. Create/update concept pages in `secondary/concepts/`
4. Use [[wiki-links]] to connect related pages
5. Update `secondary/index.md` and `secondary/log.md`

### Query Workflow
1. Read `secondary/index.md` first to find relevant pages
2. Synthesize answer from wiki pages
3. File back good analysis as new pages in `secondary/`

### Wiki Page Format
- YAML Frontmatter at top of each page:
  ```yaml
  title: "Page Title"
  sources:
    - "primary/papers/01_Codex_2021.pdf"
  related:
    - "[[linked_concept_page]]"
  last_compiled: 2026-04-09
  ```
- Frontmatter values with special characters (`[[`, `()`, `,`) must be quoted for Dataview compatibility
- All claims must cite source papers

## Academic Rules

### 1. No Hallucinations
- Do not fabricate claims, inflate results, or generate text not supported by source papers.
- Every factual claim must be traceable to a specific source paper.

### 2. Data Protection
- **`primary/` Folder**: NEVER edit or delete files in this directory.
- **`secondary/` Data**: Use as evidence. Update only through ingest/query workflows.

### 3. Manuscript Editing
- Major structural changes to `main.tex` require prior discussion and user confirmation.
- Small edits (typo fixes, citation additions) can be done directly.

### 4. Citation Management
- Check for duplicate Citation Keys in `references.bib` before adding new entries.
- Every `\cite{}` in `main.tex` must have a corresponding entry in `references.bib`.
- Every entry in `references.bib` should be cited at least once in `main.tex`.

### 5. LaTeX Quality
- Compile with: `pdflatex main.tex && bibtex main && pdflatex main.tex && pdflatex main.tex`

### 6. Comparison Table
- `secondary/paper_comparison.md` is the Single Source of Truth (SSOT).
- When converting to LaTeX table, ensure all values match the SSOT exactly.
