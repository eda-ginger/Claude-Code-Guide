# AI 에이전트와 Claude Code

AI 챗봇과 에이전트는 근본적으로 다릅니다. 이 문서에서는 그 차이를 이해하고, Claude Code를 설치하여 첫 에이전트 경험을 시작합니다.

---

## 1. 챗봇과 에이전트의 차이

### 챗봇: 한 번에 한 턴

기존 AI 챗봇은 **사람이 매번 개입**해야 합니다. 질문 → 답변 → 복사 → 붙여넣기 → 다시 질문. 사람이 모든 단계를 수동으로 연결하는 구조입니다.

```
[사람] 질문 → [AI] 답변 → [사람] 복사+붙여넣기 → [사람] 다시 질문 → ...
```

### 에이전트: 스스로 루프를 돈다

에이전트는 목표만 주어지면 **Plan → Action → 결과 확인**을 스스로 반복합니다.

```
[사람] 목표 제시
   ↓
[에이전트] 계획 수립 (Plan)
   ↓
[에이전트] 도구 사용 — 파일 읽기, 코드 수정, 명령 실행 (Action)
   ↓
[에이전트] 결과 확인 — 성공? 실패? (Observe)
   ↓
[에이전트] 다음 단계 결정 → 다시 Action... (Loop)
   ↓
[에이전트] 완료 보고
```

Claude Code가 바로 이 에이전트입니다. 파일을 읽고, 코드를 수정하고, 터미널 명령을 실행하고, 결과를 확인한 뒤 스스로 다음 행동을 결정합니다.

---

## 2. AI를 사용하는 방식의 변화

에이전트가 등장하면서, AI를 사용하는 방식도 함께 진화해왔습니다.

### 1단계: 프롬프트 엔지니어링 — "말을 잘 걸기"

AI에게 **구체적이고 상세하게 명령**을 내리는 것에 집중하던 시기입니다.

```
❌ "웹사이트 만들어줘"
✅ "모바일 반응형 + 다크모드 지원하는 로그인 페이지를 만들어줘"
```

하지만 아무리 말을 잘 걸어도, AI가 현재 프로젝트의 코드 구조나 기술 스택을 모르면 엉뚱한 결과를 만듭니다. 그래서 다음 단계가 나왔습니다.

### 2단계: 컨텍스트 엔지니어링 — "배경을 알려주기"

명령뿐 아니라 **프로젝트의 배경 정보를 함께 제공**하는 방식입니다. "우리 프로젝트는 이런 구조고, 이런 규칙을 쓰고 있어"라고 알려주는 것이죠. `CLAUDE.md`로 프로젝트 규칙을 적어주고, 스킬로 작업 가이드를 주는 것이 여기에 해당합니다.

하지만 시간이 지나면서 가이드와 도구가 수백 개로 쌓이자, 정보가 너무 많아진 AI가 오히려 헷갈리기 시작합니다.

### 3단계: 하네스 엔지니어링 — "환경을 설계하기"

정보를 계속 추가하는 대신, **AI가 올바르게 일할 수밖에 없는 환경 자체를 설계**하는 방식입니다. 핵심 차이는 "이거 하지 마"라고 **부탁**하는 것이 아니라, 시스템으로 **강제**한다는 점입니다.

| 관점 | 부탁 (프롬프트) | 강제 (하네스) |
|------|-----------------|---------------|
| 예시 | "테스트 없이 커밋하지 마" | Hook으로 테스트 실패 시 커밋 차단 |
| 결과 | AI가 잊으면 끝 | 시스템이 막으므로 실수 불가능 |

이 가이드에서 다루는 `CLAUDE.md`, 스킬, 메모리, Hooks는 모두 이런 환경 설계의 구성 요소입니다. 구체적인 설정 방법은 [02-project.md](./02-project.md)에서 다룹니다.

## 3. 에이전트가 잘하는 것과 못하는 것

### 잘하는 것

- **파편화된 자료 통합**: 형태가 제각각인 엑셀, 문서, 포스터 등을 하나의 프로젝트로 엮어 정리
- **듬성듬성한 데이터로 스토리 텔링**: 불완전한 데이터도 연결하여 스토리를 구성
- **지치지 않는 반복 시도**: 연구가 막혔을 때 참고 자료를 읽고 다음 단계 아이디어를 도출
- **API 자동화와 오류 처리**: Rate limiting, 재시도, 에러 핸들링을 알아서 처리

### 못하는 것

- **과학적 엄밀한 검증**: 통계 분석의 전제 조건을 무시하고 일단 적용. 99%만 변환되어도 "훌륭하다"며 넘어감
- **근본 오류 역추적**: 결과가 100%로 나와 명백히 설계 오류(데이터 누수 등)가 의심되어도 처음으로 돌아가지 않음
- **전진 본능**: 완벽한 처리보다 그럴싸한 결과를 만들며 어떻게든 다음 단계로 넘어가려는 성향이 강함
- **암묵적 가정의 해석**: 학계에서 통용되는 암묵지 대신, 인터넷에서 주워들은 방식으로 임의의 룰을 정하고 진행

> 에이전트는 **"그럴싸하게 전진하는 것"**에 최적화되어 있습니다. **"정확하게 멈추는 것"**은 사람의 몫입니다.

## 4. 연구자가 키워야 할 능력

| 능력 | 설명 |
|------|------|
| **작업 분할** | AI가 실수를 줄이도록, 사람이 검토 가능한 작은 단위로 쪼개어 지시 |
| **안목과 브레이크** | 계기판을 보며 "쎄하다" 싶을 때 제때 멈추는 통찰력 |
| **역추적 결단력** | 오류 위에 결과를 쌓지 않고, 과감하게 되돌려서 다시 시작하는 결단 |
| **맥락 관리** | 취할 것과 버릴 것을 엄격히 가려내는 컨텍스트 엔지니어링 |
| **향상심** | AI 결과물 중 모르는 10%를 공부해 내 것으로 소화. 소화 불가능하면 맹신하지 않고 전문가와 협업 |

> 효율이 떨어지더라도 본인 훈련을 위해 직접 머리를 써야 할 부분은 스스로 처리하세요. 무조건적인 오프로딩은 연구자 본인의 성장을 저해합니다.

## 5. Claude Code 설치

### 사전 요구사항

- **Node.js 18 이상**: `node -v`로 확인
- **Anthropic 구독**: Pro 또는 Max 플랜 (API 키도 가능)

### Node.js 설치 (미설치 시)

**[방법 A] 공식 홈페이지**
1. [nodejs.org](https://nodejs.org/)에서 **LTS 버전** 다운로드
2. 설치 프로그램 실행 (npm 자동 포함)

**[방법 B] NVM (버전 관리용)**
```bash
nvm install lts
nvm use lts
```

> ⚠️ 설치 후 **터미널을 새로 열어야** `npm` 명령어가 동작합니다.

### 설치 및 인증

```bash
npm install -g @anthropic-ai/claude-code
claude
```

1. 브라우저가 열리며 Anthropic 인증 페이지로 이동
2. 계정 로그인 → CLI 접근 권한 허용
3. 터미널에 `>` 프롬프트가 뜨면 준비 완료

## 6. 기본 사용법

### 코드 작성

```
> Python으로 피보나치 수열을 계산하는 함수를 만들어줘
```

### 코드 읽기 및 질문

```
> src/main.py에서 calculate 함수가 어떤 역할을 하는지 설명해줘
```

### 코드 수정

```
> app.js의 에러 처리를 try-catch로 감싸줘
```

### 슬래시 명령어

| 명령어 | 기능 |
|--------|------|
| `/help` | 전체 명령어 안내 |
| `/clear` | 대화 컨텍스트 초기화 |
| `/compact` | 컨텍스트 정리 (토큰 절약) |
| `/cost` | 현재 토큰 비용 확인 |

### 단축키

| 단축키 | 기능 |
|--------|------|
| `Ctrl + C` | 현재 응답 중단 |
| `Ctrl + D` | Claude Code 종료 |
| `Shift + Tab` | Plan Mode 전환 |

---

> 💡 **Tip:** 에이전트에게 지시할 때는 "이 코드 고쳐줘"보다 **"이 함수에서 null 체크를 추가해줘"**처럼 구체적으로. 그리고 결과는 항상 직접 확인하세요 — 에이전트는 실행자이고, 판단은 사람의 몫입니다.
