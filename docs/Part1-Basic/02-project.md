# 프로젝트 설정

에이전트는 매 세션마다 맥락을 잃습니다. 프로젝트에서 에이전트를 잘 활용하려면 컨텍스트 관리를 위한 준비가 필요합니다.

---

## 1. 왜 프로젝트 설정이 필요한가

에이전트는 새 세션을 시작하면 **어제 무엇을 했는지, 이 프로젝트가 어떤 규칙을 쓰는지 전부 모릅니다.** 이것이 컨텍스트 오염(Context Decay)입니다.

```
대화 시작:  "완벽하게 이해하고 있어요!"
30분 후:    "...뭘 하고 있었죠?"
다음 세션:  "이 프로젝트가 뭔가요?"
```

매번 처음부터 설명하는 대신, 에이전트가 **자동으로 읽는 파일들**을 미리 준비해두면 이 반복이 사라집니다.

---

## 2. CLAUDE.md — 프로젝트 지침서

`CLAUDE.md`는 에이전트가 **매 세션 시작 시 가장 먼저 읽는 파일**입니다. 프로젝트의 규칙, 구조, 빌드 명령어 등을 적어둡니다.

### 포함할 것과 제외할 것

| 포함 (에이전트가 추측 불가능한 것) | 제외 (코드를 읽으면 아는 것) |
|-----------------------------------|------------------------------|
| 빌드/테스트 명령어 | 표준 언어 규칙 |
| 기본값과 다른 코드 스타일 | 코드에서 파악 가능한 구조 |
| Git 워크플로 (브랜치, PR 규칙) | 상세한 API 문서 (링크로 대체) |
| 프로젝트 고유의 주의사항 | 자주 변경되는 정보 |

> 각 줄에 대해 물어보세요: **"이것을 빼면 에이전트가 실수할까?"** 아니면 삭제.

### 계층 구조

CLAUDE.md는 여러 위치에 둘 수 있으며, 상위에서 하위로 모두 연결되어 적용됩니다.

| 위치 | 적용 범위 | 용도 |
|------|-----------|------|
| `~/.claude/CLAUDE.md` | 모든 프로젝트 공통 | 개인 스타일, 선호도 |
| `프로젝트/CLAUDE.md` | 해당 프로젝트 전체 | 팀 공유 규칙 (Git 커밋) |
| `프로젝트/CLAUDE.local.md` | 해당 프로젝트 (개인) | 로컬 환경 (.gitignore 대상) |
| `프로젝트/src/CLAUDE.md` | src 작업 시 추가 적용 | 영역별 추가 규칙 (온디맨드 로드) |

### 자동 생성

```
> /init
```

에이전트가 프로젝트를 분석하여 초기 CLAUDE.md를 자동 생성합니다.

### 작성 원칙

- **200줄 이하**로 유지 — 길어지면 에이전트가 중요한 규칙을 무시하기 시작
- 길어질 경우 `@path/to/file`로 다른 파일을 임포트하여 **모듈화**
- 지시 간 **모순 방지** — 정기적으로 리뷰하여 충돌하는 규칙 삭제
- 중요한 지시는 `IMPORTANT`, `YOU MUST` 등으로 강조

### 주의: CLAUDE.md는 "부탁"이다

CLAUDE.md는 시스템 설정이 아니라 **대화 맥락(Context)**으로 주입됩니다. 에이전트가 100% 따른다는 보장이 없습니다. 반드시 지켜야 하는 규칙은 뒤에서 다루는 **Hook**으로 강제해야 합니다.

## 3. 메모리와 핸드오프 — 세션 간 기억

### 메모리: 영구 기억

에이전트에게 "이것을 기억해"라고 하면 세션이 바뀌어도 유지됩니다.

```
> 이 프로젝트에서는 항상 TypeScript strict 모드를 사용해야 한다는 걸 기억해줘
```

메모리는 `~/.claude/` 아래에 마크다운 파일로 저장되며, 에이전트가 자동으로 읽습니다. CLAUDE.md와 마찬가지로 컨텍스트로 주입되는 "부탁" 성격입니다.

### 핸드오프: 세션 인수인계서

하루 작업을 끝낼 때, 다음 세션에서 이어서 할 수 있도록 상태를 기록합니다.

```
> 오늘 작업한 내용을 정리해줘:
> 1. 완료한 작업  2. 진행 중인 작업  3. 다음에 할 작업  4. 주의사항
```

다음 세션 시작 시 이 인수인계서를 붙여넣으면, 에이전트가 즉시 이전 맥락을 복원합니다.

## 4. .claudeignore — 불필요한 파일 제외

에이전트가 읽지 않을 파일/폴더를 지정합니다. `.gitignore`와 같은 문법입니다.

```
node_modules/
dist/
*.min.js
.env
secrets/
```

불필요한 파일을 제외하면 에이전트가 핵심 코드에만 집중할 수 있고, **토큰 소비가 60~90% 줄어듭니다.**

---

## 5. 스킬 — 반복 작업의 SOP

스킬은 자주 하는 작업을 **마크다운 파일로 정의**해두고 슬래시 명령으로 실행하는 기능입니다. 사람으로 치면 업무 매뉴얼(SOP)과 같습니다. CLAUDE.md와 마찬가지로 "부탁"이지만, 구체적인 절차가 명시되어 있어 따르는 확률이 높습니다.

### 설정 방법

`.claude/commands/`에 마크다운 파일을 만들면 그 파일명이 슬래시 명령이 됩니다.

```
.claude/
├── commands/    ← 단순 프롬프트 (1파일)
│   ├── deploy.md      →  /deploy
│   └── test-all.md    →  /test-all
└── skills/      ← 복잡한 워크플로 (에이전트 정의 포함)
    └── session-pack/
        └── SKILL.md
```

**예시: `.claude/commands/deploy.md`**

```markdown
배포 전 체크리스트를 수행합니다:
1. 모든 테스트를 실행하고 통과하는지 확인
2. 빌드가 성공하는지 확인
3. 변경사항을 main 브랜치에 머지
4. 배포 스크립트를 실행
```

> `.claude/commands/`와 `.claude/skills/` 폴더를 Git에 커밋하면 팀원 모두가 같은 명령어를 사용할 수 있습니다.

## 6. MCP — 외부 도구 연결

MCP(Model Context Protocol)는 에이전트가 **외부 서비스에 접근할 수 있게 해주는 연결 통로**입니다. 부탁이나 강제가 아닌 **능력 확장**에 해당합니다.

```
[에이전트] ←→ [MCP 서버] ←→ [외부 서비스: NotebookLM, 텔레그램, GitHub, DB ...]
```

### 설정 방법

`settings.json`에 MCP 서버를 등록합니다:

```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "ghp_xxxxxxxxxxxx"
      }
    }
  }
}
```

또는 Claude Code 내에서 `/mcp` 명령으로 대화형 설정도 가능합니다.

## 7. Hook — 자동 실행 장치

Hook은 **특정 시점에 자동으로 실행되는 스크립트**입니다. CLAUDE.md가 "부탁"이라면, Hook은 **"강제"**입니다.

### 주요 Hook 시점

| Hook | 실행 시점 | 예시 |
|------|-----------|------|
| `SessionStart` | 세션 시작 시 | 지식 파일 자동 로드 |
| `PreToolUse` | 도구 실행 전 | 위험한 명령 차단 |
| `PostToolUse` | 도구 실행 후 | 린터 자동 실행 |
| `PreCompact` | 컨텍스트 압축 전 | 대화 요약 저장 |
| `SessionEnd` | 세션 종료 시 | 인수인계 자동 생성 |

### 설정 방법

`settings.json`에 정의합니다:

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "command": "npm run lint --fix $CLAUDE_FILE_PATH"
      }
    ]
  }
}
```

`matcher`를 생략하면 모든 도구 호출에 대해 실행됩니다.

### 부탁 vs 강제

| 방식 | 예시 | 결과 |
|------|------|------|
| CLAUDE.md (부탁) | "커밋 전에 테스트를 실행해줘" | 잊을 수 있음 |
| Hook (강제) | PreToolUse에서 테스트 실패 시 차단 | 시스템이 막으므로 실수 불가능 |

---

> 💡 **Tip:** 처음부터 완벽하게 설정할 필요 없습니다. `/init`으로 CLAUDE.md를 만들고, 에이전트가 실수할 때마다 규칙을 하나씩 추가하세요. 시간이 지나면 프로젝트에 맞는 하네스가 자연스럽게 만들어집니다. 이제 실제 세션에서의 조작법과 작업 패턴은 [03-session.md](./03-session.md)에서 다룹니다.
