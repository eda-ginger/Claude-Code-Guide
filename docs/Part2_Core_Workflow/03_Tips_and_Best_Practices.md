# 실전 팁과 모범 사례

이 문서에서는 Claude Code를 최대한 효과적으로 활용하기 위한 **프롬프트 작성법, 세션 관리, 자동화 패턴, 흔한 실수 회피법**을 안내합니다.

---

## 핵심 원칙

Claude Code는 단순한 챗봇이 아니라 **에이전트 코딩 환경**입니다. 파일을 읽고, 명령을 실행하고, 변경을 수행하며, 자율적으로 문제를 해결합니다.

> **컨텍스트 윈도우가 빠르게 채워지고, 채워질수록 성능이 저하됩니다.**
> 컨텍스트 윈도우는 관리해야 할 가장 중요한 리소스입니다.

---

## 1. 효과적인 프롬프트 작성법

### 구체적으로 요청하기

| 비효율적 | 효과적 |
|----------|--------|
| "이 코드 고쳐줘" | "login.js의 handleSubmit에서 빈 이메일일 때 에러 메시지를 표시해줘" |
| "테스트 만들어줘" | "UserService.create 메서드에 대해 유효/무효 입력 케이스의 단위 테스트를 작성해줘" |
| "성능 개선해줘" | "getUsers API가 N+1 쿼리 문제가 있는 것 같은데 확인하고 수정해줘" |

### 단계적으로 요청하기

큰 작업은 한 번에 요청하기보다 단계별로 나누면 품질이 높아집니다.

```
# 1단계: 현황 파악
> 현재 인증 시스템의 구조를 분석해줘

# 2단계: 계획 수립
> OAuth 2.0으로 전환하려면 어떤 파일들을 수정해야 하는지 계획을 세워줘

# 3단계: 구현
> 계획대로 1번 항목부터 구현해줘
```

### 컨텍스트 제공하기

```
> 우리 팀은 React + TypeScript를 사용하고, 상태 관리는 Zustand를 쓰고 있어.
> 사용자 프로필 편집 페이지를 만들어줘.
```

### 풍부한 콘텐츠 제공하기

- **`@`로 파일 참조**: 코드 위치를 설명하는 대신 직접 참조
- **이미지 붙여넣기**: 프롬프트에 복사/붙여넣기 또는 드래그 앤 드롭
- **URL 제공**: 문서 및 API 참조 링크 전달
- **데이터 파이프**: `cat error.log | claude`로 파일 내용 직접 전송

## 2. Claude에게 작업을 검증할 방법 제공하기

> **이것이 할 수 있는 가장 영향력 있는 단일 작업입니다.**

| 전략 | 비효율적 | 효과적 |
|------|----------|--------|
| **검증 기준 제공** | "이메일 검증 함수를 구현하세요" | "validateEmail 함수를 작성하세요. 테스트: `user@example.com`은 true, `invalid`는 false. 구현 후 테스트를 실행하세요" |
| **UI 시각적 검증** | "대시보드를 더 좋게 보이게 하세요" | "[스크린샷 붙여넣기] 이 디자인을 구현하세요. 결과의 스크린샷을 찍고 원본과 비교하세요" |
| **근본 원인 해결** | "빌드가 실패하고 있습니다" | "빌드가 이 오류로 실패합니다: [오류]. 수정하고 빌드가 성공하는지 확인하세요. 오류를 억제하지 마세요" |

## 3. 먼저 탐색하고, 계획하고, 그 다음 코드 작성하기

**Plan Mode**(`Shift+Tab`으로 전환)를 사용하여 탐색을 실행과 분리하세요.

**1단계: 탐색** (Plan Mode)
```
/src/auth를 읽고 세션 및 로그인을 어떻게 처리하는지 이해하세요.
```

**2단계: 계획** (Plan Mode)
```
Google OAuth를 추가하고 싶습니다. 어떤 파일을 변경해야 합니까? 계획을 작성하세요.
```

**3단계: 구현** (Normal Mode로 전환)
```
계획에서 OAuth 흐름을 구현하세요. 콜백 핸들러에 대한 테스트를 작성하고,
테스트를 실행하고 실패를 수정하세요.
```

**4단계: 커밋**
```
설명적인 메시지로 커밋하고 PR을 열기
```

> 범위가 명확하고 수정이 작은 작업(오타 수정, 로그 추가 등)에는 계획이 필요 없습니다.

## 4. 효과적인 CLAUDE.md 작성하기

> `/init`을 실행하여 시작 CLAUDE.md를 생성한 다음 시간이 지남에 따라 개선하세요.

### 포함 vs 제외

| 포함할 것 | 제외할 것 |
|-----------|-----------|
| Claude가 추측할 수 없는 Bash 명령 | Claude가 코드를 읽어서 파악할 수 있는 것 |
| 기본값과 다른 코드 스타일 규칙 | Claude가 이미 아는 표준 언어 규칙 |
| 테스트 지시사항 및 선호하는 러너 | 상세한 API 문서 (대신 링크) |
| 저장소 에티켓 (분기 이름, PR 규칙) | 자주 변경되는 정보 |

> 각 줄에 대해 물어보세요: *"이것을 제거하면 Claude가 실수를 할까?"* 그렇지 않으면 삭제하세요.

### CLAUDE.md 위치별 적용 범위

| 위치 | 적용 범위 |
|------|-----------|
| `~/.claude/CLAUDE.md` | 모든 프로젝트에 공통 적용 |
| `./CLAUDE.md` | 해당 프로젝트 전체 (Git 커밋 대상) |
| `./하위디렉토리/CLAUDE.md` | 해당 디렉토리 작업 시 추가 적용 |

## 5. 세션 관리하기

### 조기에 자주 방향 수정하기

| 단축키 | 기능 |
|--------|------|
| `Esc` | Claude의 작업을 중간에 중지 (컨텍스트 보존) |
| `Esc + Esc` 또는 `/rewind` | 이전 대화 및 코드 상태 복원 |
| `"Undo that"` | Claude에게 변경 사항 되돌리기 요청 |
| `/clear` | 컨텍스트 완전 초기화 |

> 한 세션에서 같은 문제에 대해 **두 번 이상 수정**했다면, `/clear`를 실행하고 더 나은 프롬프트로 새로 시작하세요.

### Subagent로 조사하기

> 컨텍스트가 기본 제약 조건이므로 subagent는 가장 강력한 도구 중 하나입니다.

```
subagent를 사용하여 인증 시스템이 토큰 새로 고침을 어떻게 처리하는지 조사하세요.
```

### 대화 재개하기

```bash
claude --continue    # 가장 최근 대화 재개
claude --resume      # 최근 대화에서 선택
```

### Claude가 당신을 인터뷰하도록 하기

큰 기능의 경우, Claude에게 인터뷰를 요청하세요:

```
[간단한 설명]을 빌드하고 싶습니다. AskUserQuestion 도구를 사용하여 자세히 인터뷰해주세요.

기술 구현, UI/UX, 엣지 케이스, 트레이드오프에 대해 질문하세요.
모든 것을 다룰 때까지 인터뷰를 계속한 다음 SPEC.md에 완전한 사양을 작성하세요.
```

## 6. 자주 쓰는 활용 패턴

### 탐색 → 이해 → 수정 패턴

```
> 이 프로젝트의 전체 구조와 주요 진입점을 알려줘    # 탐색
> 결제 처리 로직이 어떻게 동작하는지 설명해줘       # 이해
> 결제 실패 시 재시도 로직을 추가해줘              # 수정
```

### 테스트 주도 수정 패턴

```
> 현재 실패하는 테스트를 보여줘
> 실패 원인을 분석해줘
> 테스트가 통과하도록 코드를 수정해줘
> 다시 테스트를 실행해서 확인해줘
```

### 코드 리뷰 패턴

```
> 이 PR의 변경사항을 보안 관점에서 리뷰해줘
> 성능 이슈가 있는지도 확인해줘
> 발견된 문제점에 대한 수정을 제안해줘
```

## 7. 자동화 및 확장하기

### 비대화형 모드

```bash
# 일회성 쿼리
claude -p "이 프로젝트가 무엇을 하는지 설명하세요"

# 구조화된 출력
claude -p "모든 API 엔드포인트 나열" --output-format json

# 실시간 스트리밍
claude -p "이 로그 파일 분석" --output-format stream-json
```

### 여러 세션 병렬 실행 — Writer/Reviewer 패턴

| 세션 A (작성자) | 세션 B (검토자) |
|----------------|----------------|
| `속도 제한기 구현` | |
| | `rateLimiter.ts를 검토하세요. 엣지 케이스, 경쟁 조건을 찾으세요` |
| `검토 피드백을 반영하세요: [세션 B 출력]` | |

### 파일 전체에 Fan Out

```bash
# 1. 마이그레이션 대상 파일 목록 생성
claude -p "마이그레이션이 필요한 모든 Python 파일 나열" > files.txt

# 2. 루프로 병렬 처리
for file in $(cat files.txt); do
  claude -p "React에서 Vue로 $file 마이그레이션. OK 또는 FAIL 반환." \
    --allowedTools "Edit,Bash(git commit *)"
done
```

## 8. 흔한 실패 패턴 피하기

| 실패 패턴 | 증상 | 해결 방법 |
|-----------|------|-----------|
| **주방 싱크 세션** | 한 세션에서 관련 없는 여러 작업을 섞음 | 관련 없는 작업 간에 `/clear` |
| **반복적 수정** | 수정하고, 또 잘못하고... 컨텍스트가 오염 | 두 번 실패 후 `/clear`하고 더 나은 프롬프트로 재시작 |
| **과도한 CLAUDE.md** | 너무 길어서 Claude가 중요한 규칙을 무시 | 무자비하게 정리. Claude가 이미 올바르게 하는 것은 삭제 |
| **신뢰-검증 간격** | 그럴듯하지만 엣지 케이스 미처리 | 항상 검증(테스트, 스크립트, 스크린샷) 제공 |
| **무한 탐색** | 범위 없이 "조사하세요" 요청 | 좁게 범위 지정하거나 subagent 사용 |

## 10. 트러블슈팅

### "Permission denied" 에러

```bash
claude --allowedTools Bash,Read,Edit
```

### 컨텍스트 초과

- `/compact`로 압축하거나 `/clear` 후 필요한 맥락만 다시 전달

### Claude Code 업데이트

```bash
npm update -g @anthropic-ai/claude-code
```

### 네트워크 문제

```bash
export HTTPS_PROXY=http://proxy.company.com:8080
```
